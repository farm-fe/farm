# @farmfe/plugin-vue

Compile [Vue 3 Single-File Components](https://vuejs.org/guide/scaling-up/sfc.html) using the [`fervid`](https://github.com/phoenix-ru/fervid) Rust compiler.

This plugin is the Farm-native equivalent of [`unplugin-vue`](https://github.com/unplugin/unplugin-vue) / [`@vitejs/plugin-vue`](https://github.com/vitejs/vite-plugin-vue), but written entirely in Rust — no JavaScript orchestration is required at compile time.

:::caution Experimental
`fervid`'s SFC compiler is still maturing (the upstream README marks the compile entry point as "not production-ready yet"). Expect rough edges on edge cases such as advanced `<script setup>` macros and SSR codegen. Please file issues with reproductions; we'll forward to the upstream as needed.
:::

## Usage

`@farmfe/plugin-vue` is a Rust plugin — register it in the `plugins` array of your `farm.config.ts`.

```ts
import { defineConfig } from "@farmfe/core";

export default defineConfig({
  plugins: [
    ["@farmfe/plugin-vue", {
      // options (see below) — may be omitted entirely
    }],
  ],
});
```

If you don't need any options the bare package name works too:

```ts
export default defineConfig({
  plugins: ["@farmfe/plugin-vue"],
});
```

## Options

| Option                                       | Type                                                  | Default        | Description                                                                                                                                              |
| -------------------------------------------- | ----------------------------------------------------- | -------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `include`                                    | `string \| RegExp \| (string \| RegExp)[]`            | `/\.vue$/`     | Files matched by this filter are treated as Vue SFCs.                                                                                                    |
| `exclude`                                    | `string \| RegExp \| (string \| RegExp)[]`            | `[]`           | Files excluded from the Vue SFC pipeline.                                                                                                                |
| `isProduction`                               | `boolean`                                             | derived        | Force production-mode codegen regardless of Farm's compilation mode.                                                                                     |
| `ssr`                                        | `boolean`                                             | `false`        | Enable fervid's SSR codegen path. Experimental.                                                                                                          |
| `sourceMap`                                  | `boolean`                                             | `true`         | Request source maps from fervid.                                                                                                                         |
| `features.optionsAPI`                        | `boolean`                                             | `true`         | Drives the `__VUE_OPTIONS_API__` runtime define flag.                                                                                                    |
| `features.prodDevtools`                      | `boolean`                                             | `false`        | Drives `__VUE_PROD_DEVTOOLS__`.                                                                                                                          |
| `features.prodHydrationMismatchDetails`      | `boolean`                                             | `false`        | Drives `__VUE_PROD_HYDRATION_MISMATCH_DETAILS__`.                                                                                                        |
| `features.propsDestructure`                  | `boolean`                                             | fervid default | Enable reactive destructure for `defineProps`.                                                                                                           |
| `features.customElement`                     | `boolean \| string \| RegExp \| (string \| RegExp)[]` | `/\.ce\.vue$/` | Files matching this pattern are compiled in custom-element mode.                                                                                         |

### Auto-injected `define` flags

In the `config` hook the plugin populates `compilation.define` with the standard Vue feature flags if you have not set them yourself:

- `__VUE_OPTIONS_API__` &nbsp;→ `true`
- `__VUE_PROD_DEVTOOLS__` &nbsp;→ `false`
- `__VUE_PROD_HYDRATION_MISMATCH_DETAILS__` &nbsp;→ `false`

`vue` is also added to `resolve.dedupe` (except when the build targets Node).

### Style sub-blocks

Each `<style>` block is exposed as a virtual sub-module with the same query format Vite/Rollup users expect:

```
foo.vue?vue&type=style&idx=<N>&lang=<lang>&scoped=<bool>
```

For native CSS the virtual module is emitted as `ModuleType::Css` and handled by Farm's built-in CSS pipeline. For non-CSS languages (`scss`, `sass`, `less`, `stylus`) it is emitted as `ModuleType::Custom("<lang>")` so a downstream preprocessor plugin (e.g. `@farmfe/plugin-sass`) can claim it.

## Feature support (Phase A)

The plugin's v0 surface intentionally covers a subset of `unplugin-vue` — anything not listed here is on the Phase B roadmap.

| Capability                                                | Status |
| --------------------------------------------------------- | ------ |
| `.vue` load + main module transform                       | ✅      |
| `<script setup>` / Options API                            | ✅      |
| Scoped CSS for native `<style>` blocks                    | ✅      |
| `v-bind(var)` CSS variables (native CSS)                  | ✅      |
| Custom elements via `.ce.vue` or `features.customElement` | ✅      |
| `__VUE_*__` define flag injection                         | ✅      |
| `<style lang="scss">` etc. handed off to CSS preprocessor | ✅      |
| Granular HMR (template-only / style-only)                 | Phase B (currently coarse — full module reload) |
| Preprocessor re-scoping (scope id injected post-sass)     | Phase B |
| Custom blocks (`<i18n>`, …) executor hooks                | Phase B |
| `inlineTemplate: false` / external template split         | Phase B |
| Type-only `defineProps<…>()` cross-file deps              | Phase B |
| `componentIdGenerator`                                    | Upstream fervid |
| Custom `compiler` (pin `vue/compiler-sfc` version)        | Dropped — fervid is the compiler |

## Comparison to `unplugin-vue` / `@vitejs/plugin-vue`

`@farmfe/plugin-vue` ports the bundler-facing behaviour of `unplugin-vue`'s Farm hooks (define flag injection, virtual style modules, custom-element filter) to Rust while delegating actual SFC compilation to `fervid` instead of `vue/compiler-sfc`. Most options carry over with the same names; the table above lists the ones that intentionally do not.
