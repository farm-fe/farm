# @farmfe/js-plugin-react-compiler

JavaScript plugin that runs React Compiler through Babel for matched JSX/TSX modules.

## Installation

<Tabs>
  <TabItem value="npm" label="npm">
    <CodeBlock>npm install @farmfe/js-plugin-react-compiler --save-dev</CodeBlock>
  </TabItem>
  <TabItem value="yarn" label="yarn">
    <CodeBlock>yarn add @farmfe/js-plugin-react-compiler --dev</CodeBlock>
  </TabItem>
  <TabItem value="pnpm" label="pnpm">
    <CodeBlock>pnpm add @farmfe/js-plugin-react-compiler --save-dev</CodeBlock>
  </TabItem>
</Tabs>

## Usage

```ts
import { defineConfig } from "@farmfe/core";
import react from "@farmfe/plugin-react";
import { reactCompiler } from "@farmfe/js-plugin-react-compiler";

export default defineConfig({
  plugins: [
    reactCompiler(),
    react(),
  ],
});
```

## Options

- `filters?: { moduleTypes?: ModuleType[]; resolvedPaths?: string[] }` — modules to transform. Defaults to module types `["jsx", "tsx"]` and no path filters.
- `transformOptions?: TransformOptions` — declared in the public TypeScript interface.
- `compilerOptions?: Partial<PluginOptions>` — declared in the public TypeScript interface.

The generated Farm plugin is named `"js-plugin:react-compiler"` and uses priority `120`.

:::note
The current implementation always configures Babel with `babel-plugin-react-compiler` and `@babel/plugin-syntax-jsx`. Although `transformOptions` and `compilerOptions` are typed, they are not currently forwarded into the Babel options.
:::
